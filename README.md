# HQI Dashboard

Notion의 HQI 문서, 테스트 케이스 DB, 결함 현황 DB를 기준으로 테스트 프로젝트별 HQI 점수를 자동 계산하는 로컬 웹 프로그램입니다.

## 실행

```bash
export NOTION_TOKEN="Notion integration token"
python3 app.py
```

브라우저에서 `http://127.0.0.1:8000`을 엽니다.

노션 임베드에는 프로젝트별 품질 점수 테이블과 5.18~5.21 정기 업데이트 버전별 점수 그래프만 표시하는 `http://127.0.0.1:8000/embed.html`을 사용합니다. 이 페이지는 `embed-data.json`을 읽어오며, `수동 갱신` 버튼으로만 다시 읽습니다. 데이터가 바뀌면 `embed-data.json`을 다시 생성해서 함께 배포하면 됩니다.

## 사용 데이터

- 테스트 케이스
  - `신규 기능 - 정기업데이트`
  - `비정기 업데이트`
- 결함 현황
  - `QA_ISSUES`

## 적용 산식

노션 HQI 문서 기준으로 아래 산식을 적용합니다.

- `TPR = 수행 TC 수 / 전체 TC 수`
- `BOR = 1 - 결함 수 / 수행 TC 수`
- `BFR = 수정 완료 결함 수 / 전체 결함 수`
- `DQS = (BOR + BFR) / 2`
- `HQI = 1 - sqrt((TPR - 1)^2 + (DQS - 1)^2) / sqrt(2)`

프로젝트별 TC 수는 테스트 프로젝트 페이지 하위의 `child_database`를 재귀 탐색해서 계산합니다. 각 TC 행에 `TC cnt`가 있으면 해당 값을 합산하고, 없으면 행 1개를 TC 1건으로 봅니다. 수행 TC 수는 하위 TC 행의 `테스트 진행율` 또는 `진행율`을 우선 사용하고, 없으면 프로젝트의 진행율을 사용합니다.
하위 TC DB가 없는 프로젝트는 TC 컬럼에 `미집계`로 표시하고, HQI는 프로젝트 진행율과 결함 집계 기준으로 계산합니다.

상단 셀렉트 박스에서 테스트 결과서 기준 프로젝트를 선택하고 `계산`을 누르면 해당 프로젝트만 Notion 원본에서 읽어 계산합니다. `전체 프로젝트 추이 계산`을 누르면 전체 프로젝트를 순차 계산해 추이/퍼널용 저장 결과를 갱신합니다. 계산 결과는 로컬 SQLite DB(`hqi_results.sqlite3`)에 저장되며, 이후 화면은 저장된 결과를 기준으로 표시합니다. 동일 프로젝트에 추가 업데이트가 없으면 저장된 결과를 그대로 표시합니다.
상태가 `시작 전`인 프로젝트는 프로젝트 목록, 계산 대상, 저장 결과 화면에서 제외합니다.

## 화면 구성

- 프로젝트별 HQI 점수 테이블
- 노션 임베드용 프로젝트별 HQI 점수 테이블: `/embed.html`
- 프로젝트별 2차원 품질 좌표 그래프: X축 `TPR`, Y축 `DQS`
- 전체 프로젝트 HQI 등급 퍼널 그래프
- 5.18 이후 버전형 정기 업데이트 품질 추이 그래프
- 선택 프로젝트 상세: TC DB 집계, 결함 수, 심각도, 결함 링크

## 추가되면 좋은 데이터

결함 DB의 `목표버전`과 테스트 프로젝트명이 직접 일치하지 않는 프로젝트는 별도 매핑 컬럼이 있으면 더 정확하게 결함이 연결됩니다.
